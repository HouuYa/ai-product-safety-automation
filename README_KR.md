## 🎯 개요

AI를 활용한 엔드투엔드 자동화 시스템:
- **43개국** 리콜 제품 정보 수집·관리
- **OECD 글로벌 리콜 포털**과 실시간 연계
- **온라인 쇼핑몰** 리콜제품 유통 및 사고징후 감시
- **뉴스 포털**에서 제품사고 실시간 탐지
- **소비자 후기·문의** 분석을 통한 조기 경보

**전문 개발자가 아닌 공무원이 직접 구축** — 현대 AI와 Low-Code 도구를 활용하면 비기술자도 엔터프라이즈급 자동화 시스템을 구축할 수 있음을 증명

### 주요 성과
- ⚡ **처리시간 83% 단축** (30분 → 5분/건)
- 💰 **연간 비용 95% 절감** (2,000만원 → 100만원)
- 🌍 **43개국** 리콜 데이터 자동 수집
- 🔄 **실시간** OECD 포털 연동
- 🤖 **AI 기반** 유통감시 및 멀티모달 분석
- 📰 **2시간 주기** 제품사고 뉴스 모니터링

## 🚀 주요 기능

### 1. 국내 리콜 → OECD 등록 자동화
국내 리콜 제품 정보를 OECD 글로벌 리콜 포털에 자동 등록

**프로세스:**
1. 제품안전정보 포털(`safetykorea.kr`) 매일 모니터링
2. API를 통한 리콜 정보 추출
3. GPT-4/Gemini로 영문 번역
4. GPC(국제 표준 제품분류) 코드 자동 부여
5. OECD 요구 XML 형식 생성
6. `globalrecalls.oecd.org`에 자동 제출

**성과:** 8년간 외주 계약 대체, 연간 1,900만원 절감

[자세히 보기 →](./docs/01-domestic-recall-oecd.md)

---

### 2. 해외 리콜 정보 수집 (43개국)
다국어 스크래핑 및 AI 분석을 통한 국제 리콜 데이터 수집

**수집 국가:** 미국, 캐나다, 일본, 중국, 호주, 뉴질랜드, EU(27개국)

**프로세스:**
1. 13개 정부 웹사이트 매일 스크래핑
2. 리콜 공표문, 이미지, PDF 추출
3. 다국어 번역 (영어, 중국어, 일본어, 독일어, 프랑스어)
4. 위해 유형 및 화학물질 분류
5. 구조화된 데이터베이스 저장
6. 중대 리콜 정보 Telegram으로 실시간 알림

**성과:** 처리 시간 4일 → 1시간 이내로 단축 (75% 감소)

[자세히 보기 →](./docs/02-overseas-recall-collection.md)

---

### 3. 온라인 쇼핑몰 유통 감시
AI 기반 리콜제품 판매 탐지 및 사고 징후 분석

**프로세스:**
1. 쿠팡, 네이버쇼핑 등 제품 정보 스크래핑 (모델번호, 이미지, 후기)
2. 리콜 데이터베이스와 교차 검증
3. GPT-4 Vision 멀티모달 분석
   - 제품 이미지 비교
   - 소비자 후기에서 사고 징후 키워드 탐지
4. 위해성 평가 보고서 생성

**성과:** 수동 검토 시간 60% 단축

[자세히 보기 →](./docs/03-marketplace-surveillance.md)

---

### 4. 제품사고 뉴스 모니터링
제품사고 뉴스 자동 탐지 및 즉시 전파

**프로세스:**
1. 네이버 뉴스, 소방방재신문 2시간 주기 모니터링
2. AI 기반 사고 분류 (화재, 폭발, 감전 등)
3. 제품 정보 및 위해 내용 추출
4. Telegram을 통해 관계자에게 즉시 전파

**성과:** 주요 제품사고 정보 실시간 공유 체계 구축

[자세히 보기 →](./docs/04-news-monitoring.md)

## 🛠 기술 스택

| 계층 | 기술 | 용도 |
|-------|-----------|---------|
| **자동화** | n8n (cloud) | 워크플로우 오케스트레이션 |
| **AI/ML** | Gemini 2.0 Flash, GPT-4, GPT-4 Vision | 번역, 분류, 분석 |
| **데이터베이스** | Supabase (PostgreSQL) | 구조화된 데이터 저장 |
| **스크래핑** | ScrapingBee, Puppeteer | 웹 데이터 추출 |
| **API** | REST, RSS, Webhooks | 시스템 통합 |
| **알림** | Telegram API | 실시간 알림 |

**Gemini 2.0 Flash 선택 이유:** 무료 티어에서 높은 요청 한도, 정부 예산 제약 하에서도 품질 유지

**n8n Cloud 선택 이유:** 서버 관리 불필요, 빠른 구축, 유지보수 용이
