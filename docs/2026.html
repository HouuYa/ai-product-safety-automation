과제 01
KC안전기준 디지털화
KC Safety Standard Digitization
총 2년간 4.45억 (1차년 1.97억) · 정부 R&D과제 기획 및 수행기관 선정 완료 Total ₩445M (Year 1: ₩197M) · R&D

330종 KC 안전기준을 HWP 고시 문서에서 구조화 DB로 변환하고, AI 자연어 검색 엔진을 구축
"이 제품에 어떤 기준이 적용되나요?"라는 질문에 AI가 99% 정확도로 즉시 답변할 수 있는 수준을 목표

Converts 330 KC safety standards from HWP notices into structured DB, building an AI natural language search engine. Target: AI answers "which standard applies to this product?" instantly with 99% accuracy.

1
KC 안전기준 330종 HWP → 구조화 텍스트 추출330 KC standards HWP → structured text extraction
2
조항·품목·시험방법 메타데이터 DB 설계·적재Clause, item, test method metadata DB design & load
3
RAG 기반 AI 자연어 검색 엔진 구현 (99% 검증)RAG-based AI natural language search engine build (99% verification)
4
포털 연동 API 개발 (2027 기반과제 모듈 설계)Portal integration API (designed as 2027 portal module)

과제 03
사고징후 선제포착 시스템
Proactive Accident Signal Detection
예산 1.97억 · 정부 R&D과제 기획 및 수행기관 선정 완료 Budget ₩197M · R&D

SNS, 맘카페, 소비자 게시판 등 공개 텍스트를 자연어 처리로 분석하여 리콜 공식화 이전에 사고 징후를 포착
기존 30일이 걸리던 징후 포착 소요기간을 3일로 90% 단축하는 것을 목표

Analyzes public text from SNS, parenting communities, and consumer boards via NLP to detect accident signals before official recall. Aims to reduce signal detection time by 90% (from 30 days to 3 days).

1
수집 대상: SNS·맘카페·소비자 게시판 크롤링Collection targets: SNS, parenting forums, consumer boards
2
NLP 모델(NER, 감정분석)로 위해유형 키워드 추출NLP model (NER, Sentiment) extracts hazard keywords
3
이상 급증 패턴 감지 → 조기 경보 알고리즘Anomaly surge detection → early warning algorithm
4
담당자 조기 경보 발령 + HITL(Human-In-The-Loop, 인력) 검증Early warning alert to staff + HITL verification
과제 04
PHICS 위해요인 분류체계 수립
PHICS Hazard Classification System
내부 리서치를 통해 추진 중 · ISO 5665:2024, 영국 PRISM, 일본 METI Recall guidelines Internal · ISO 5665:2024, UK PRISM, Japan METI Recall guidelines

PHICS(Product Hazard Information Classification System): 위해 원인(HF, Hazard Factor)과 피해 결과(DT, Damage Type)를 이중 코드로 분류하는 국제 표준(ISO 5665 및 영국, 일본 등) 정렬 체계
리콜 통계와 사고조사 대장의 데이터 품질을 근본적으로 개선

PHICS: Dual-code classification aligning Hazard Factor (HF, cause) and Damage Type (DT, outcome) with ISO 5665:2024 standards. Fundamentally improves recall statistics and accident investigation data quality.

1
현행 리콜 통계·사고조사 분류체계 현황 진단Diagnose current recall stats & investigation classification
2
HF(위해원인) + DT(피해결과) 이중 코드 체계 설계Design dual-code: HF (hazard factor) + DT (damage type)
3
ISO 5665:2024 국제 표준 매핑·검증Map and validate against ISO 5665:2024 international standard
4
기존 데이터 소급 적용 및 DB 마이그레이션Retroactive application to existing data & DB migration
과제 05

AI Call Center Pilot for Product Incidents
예산 0.2억 · 보조금 Budget ₩20M · Subsidy

1670-4920(2번 내선) 전화 문의의 55%를 차지하는 단순·반복 민원을 AI 챗봇이 24시간 자동 응대
개인정보 수집이 필요하거나, 복잡하거나 법적 판단이 필요한 민원만 담당자에게 연결, 실무자는 고부가가치 업무에 집중

AI chatbot handles 55% of simple/repetitive inquiries on the 1670-4920 hotline 24/7. Only complex or legally-sensitive cases are escalated to staff, freeing officials to focus on high-value work.

1
기존 콜센터 VOC 분석 — 단순문의 패턴 추출VOC analysis of existing call logs — extract simple inquiry patterns
2
FAQ 기반 LLM 챗봇 개발 (RAG 적용)FAQ-based LLM chatbot development (RAG applied)
3
24시간 자동 응대 · 처리 불가 시 담당자 연결24/7 auto-response; escalate unresolvable cases to staff
4
응대 품질 모니터링 + HITL 지속 개선Response quality monitoring + HITL continuous improvement
